    Практическое занятие 4. Запросы к данным (SELECT)

    Продолжительность: 2 академических часа (90 минут)

    Тип: практическое занятие

    Цель занятия: сформировать устойчивые навыки выборки данных из одной и нескольких таблиц; фильтрации (WHERE); сортировки (ORDER BY); ограничения выборки (LIMIT); работы с условиями (IN, BETWEEN, LIKE, IS NULL); соединения таблиц (JOIN)

    Исходная структура БД

    CREATE TABLE groups (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL UNIQUE
);

CREATE TABLE students (
    id SERIAL PRIMARY KEY,
    full_name TEXT NOT NULL,
    birth_date DATE,
    group_id INT REFERENCES groups(id),
    scholarship NUMERIC
);

CREATE TABLE teachers (
    id SERIAL PRIMARY KEY,
    full_name TEXT NOT NULL,
    department TEXT
);

    Перед началом занятия таблицы должны быть заполнены тестовыми данными (10–20 строк).

    Этап 1. Базовые выборки

    Задание 1

    Вывести всех студентов.

    Задание 2

    Вывести:

    • ФИО
    • дату рождения

    Задание 3

    Вывести студентов с стипендией больше 3000.

    Задание 4

    Вывести студентов без стипендии.

    Использовать IS NULL.

    Этап 2. Условия и логические операторы

    Задание 5

    Найти студентов:

    • из определённой группы
    • со стипендией больше 2500

    Задание 6

    Вывести студентов, родившихся:

    • между 2004 и 2006 годами.
    • Использовать BETWEEN.

    Задание 7

    Найти студентов:

    • у которых фамилия начинается на «И».
    • Использовать LIKE.

    Этап 3. Сортировка и ограничение

    Задание 8

    Вывести студентов:

    • отсортированных по стипендии по убыванию.

    Задание 9

    Вывести 5 студентов с самой высокой стипендией.

    Использовать ORDER BY + LIMIT.

    Задание 10

    Вывести 3–7 записи (использовать OFFSET).

    Этап 4. JOIN (связанные данные)

    Задание 11

    Вывести:

    • ФИО студента
    • название группы

    Использовать JOIN.

    Задание 12

    Вывести всех студентов, включая тех, у кого нет группы.

    Использовать LEFT JOIN.

    Объяснить разницу между INNER и LEFT JOIN.

    Этап 5. Комплексный запрос

    Задание 13

    Вывести:

    • ФИО студента
    • название группы
    • возраст (вычислить)
    • стипендию

    Условия:

    • стипендия больше средней по таблице
    • отсортировать по возрасту

    Индивидуальные задания

    Вариант 1. Найти студента с минимальной стипендией.

    Вариант 2. Найти студентов старше 20 лет.

    Вариант 3. Найти группы без студентов.

    Вариант 4. Найти студентов, чьё имя состоит более чем из 15 символов.

    Вариант 5. Найти студентов, у которых стипендия выше средней по группе.
